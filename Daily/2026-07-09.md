## Work & Projects

### Telegram bridge — rebuilt agentic (16:30-17:00)
Rob hit the wall: the bridge "hung" and was unusable for getting work done. Diagnosed live: it wasn't crashed (alive, polling, no errors), the design was the problem. Old bridge was one-shot chat-only: each message = one `claude -p` returning text, then the process died. So every "on it, give me a few" was a dead end (nothing ran between messages) and the 240s single-threaded loop couldn't do real work anyway.
Rebuilt `Jarvis/bridge/bridge.py`: single background worker + queue (poll loop never blocks), instant ack (typing kept alive + text ack past 8s), **agentic** runs (full tools, skip-perms, 25-min timeout) that actually do the work end to end then report back. Jobs run one at a time so two runs don't fight the same git repo. Prompt hammers "there is no later, act now." README updated. Restarted via run.sh (tmux + cron watchdog already reboot-proof). Live and polling.
