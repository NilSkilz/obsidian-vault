### Tethered: PRODUCTION 400 on saving a record (context GSI)
- Rob hit a production error saving a checklist record: `Type mismatch for Index Key context Expected: S Actual: NULL, IndexName: byOwnerAndContext`.
- Root cause: `context` is the sort key of ChecklistAnswer's `byOwnerAndContext` GSI (schema since April). Saving a Default-profile answer sends `context: null`, and AmplifyAdapter passed it straight through, so AppSync wrote a NULL-typed key attribute, which DynamoDB rejects (GSI key must be string or absent). Surfaced now because the backend was redeployed on 2026-07-13 (`2e276c1`), activating the GSI on production.
- Fix (`develop`, committed NOT pushed): create/updateAnswer omit context when null/empty (absent = default, matches `!a.context` reads); "move answers to default" on profile delete now recreates answers without context + deletes originals (a GSI key can't be nulled via update); regression test on the exact payloads. tsc clean, 436 tests pass. Verified at the adapter boundary (payload no longer contains null context), not end-to-end against live AWS.
- **NEEDS ROB: push + deploy to production.** This is a live user-facing breakage (every first-time Default-profile rating fails). No schema change or data migration needed, purely the client write payload.
